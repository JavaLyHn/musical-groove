# 连接真实音乐（macOS）

声音星球用浏览器的 Web Audio 实时分析一个**音频输入设备**的频谱来驱动柱阵。
难点在于：浏览器默认只能拿到「麦克风」，拿不到「系统正在播放的声音」。
解决办法是装一个**虚拟声卡**（BlackHole，免费开源），把系统输出引到它，
再让网页从它「录音」——这样拿到的就是你正在放的音乐。

## 一、安装 BlackHole（虚拟声卡）

```bash
brew install blackhole-2ch
```

（或到 https://existential.audio/blackhole/ 下载安装包。装完可能需要重启或在
「系统设置 → 隐私与安全性」里允许该驱动。）

## 二、建一个「多输出设备」（这样你既能听到声音，又能把它喂给网页）

1. 打开「**音频 MIDI 设置**」(Audio MIDI Setup，在 /应用程序/实用工具)。
2. 左下角 `+` → **创建多输出设备**。
3. 勾选 **你的扬声器/耳机** + **BlackHole 2ch** 两个。
4. （建议把你的扬声器设为 Master、勾上「漂移校正」。）
5. 回到「系统设置 → 声音 → 输出」，把输出选成这个**多输出设备**。
   - 现在系统声音会同时进你的耳朵 + BlackHole。

## 三、连接

1. 放一首歌（任意播放器：Apple Music / Spotify / 浏览器都行）。
2. 打开可视化页面（开发时是 `http://127.0.0.1:5188`）。
3. 点底部「**▶ 连接系统声音**」按钮，浏览器会请求麦克风权限 → 允许。
4. 页面会**自动优先选用 BlackHole** 作为输入，于是柱阵就跟着你放的音乐跳动了。

## 没装 BlackHole 会怎样？

按钮仍可用，但只能连到**默认麦克风**——这时它跟着环境/外放的声音跳动，
而不是精确同步到那一首曲子。要精确同步系统播放的音乐，就得走上面的 BlackHole 方案。

## 接口说明（给开发者）

真实音频源 `src/webAudioSource.js` 与模拟源 `src/audioSource.js` 实现**完全相同的接口**：
`getSpectrum() → Float32Array(64)`、`getLevels() → {bass,mid,treble}`、`update(dt)`。
所以视觉层一行都不用改，`main.js` 里连接后直接把 `audio` 换成真实源即可。
