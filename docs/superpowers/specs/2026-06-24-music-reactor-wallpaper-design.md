# 声音星球 · 音乐反应式动态壁纸 — 设计文档

> 日期：2026-06-24
> 状态：设计已确认，待写实现计划

---

## 1. 概述

一片贴在深空星球表面的发光柱阵——既像城市天际线，也像反应堆的燃料棒。柱子随音乐能量上下跳动，中心有一个发光"堆芯"在重拍时爆亮。最终作为 **macOS 动态壁纸**（用 Plash 这类「把网页当壁纸」的工具渲染一个全屏循环网页）。

参考图：`451771f61ca81c36105c5cbcb747e810.jpg`（曲面网格柱阵 / 冷调蓝紫 / 低视角望向地平线）。

**本期范围**：把视觉与动效做到位，用**模拟音频数据**驱动柱子跳动。真正接入系统声音、Plash 部署、设置面板等留待后续。

## 3. 体验与状态

壁纸长期挂后台，必须「永远在动、但不吵」。两种状态平滑过渡：

- **静息态（无强音 / 无音乐）**：全场用柏林噪声做缓慢起伏，相机极慢漂移。安静、有生命，不会像死图。
- **激活态（音乐播放）**：柱子随节拍跳动；低频重拍时中心堆芯爆亮 + 一圈光环向外扩散；能量像涟漪从中心荡到边缘。

两态之间没有硬切换——静息呼吸始终作为"底噪"叠加在音频响应之下。

---

## 4. 场景构图（从后到前的层次）

1. **深空背景**：`#070912 → #0d1430` 径向渐变，中心略亮形成柔光与暗角（vignette）。
2. **星点粒子**：缓慢漂浮的细小光点，营造宇宙夜空的景深。
3. **柱阵主体**（主角）：球面/穹顶上的网格柱阵。
4. **中心堆芯**：正中央的发光核心 + 地面辉光。
5. **大气雾**：远处柱子被淡雾"吃掉"，形成地平线和纵深（Three.js fog）。
6. **相机**：贴近地平面的低视角，越过曲面望向远方；极缓慢自动环绕 + 上下浮动，正弦循环、无限不穿帮。
7. **后期**：Bloom 辉光 + 轻微暗角。

---

## 5. 柱阵系统（核心）

- **网格规模**：约 `40 × 40 ≈ 1600` 根方柱（性能档位可调，见 §9）。
- **球面映射**：把平面网格用穹顶函数弯成球面，还原"城市贴在星球上"的弧度。柱子始终沿球面法线方向"立起"。
- **柱体样式**：**分段方块柱**——每根柱子由若干等高小方块堆叠而成，块间留小缝，顶面更亮、侧面更暗，呈现积木质感。块数随高度增减。
- **能量分布**：**中心高、四周低**。低频能量在中心聚集形成高耸山丘，向外逐渐变矮、变暗、变冷，形成参考图那种中央隆起的形态。

---

## 6. 配色 / 材质 / 光

- **背景**：`#070912 → #0d1430`
- **柱体主色带**：冷蓝 `#2a3a8c` → 紫 `#6a4fd0`，按高度/能量插值
- **峰值高光**：接近白的薰衣草 `#d8d0ff`（最高最亮的柱子 / 块顶）
- **材质**：自发光（emissive）为主，亮柱在 Bloom 下真正"发光"；块顶面比侧面亮，营造受光感
- **堆芯**：暖白偏冷的高强度发光体，是全场最亮点

---

## 7. 运动设计（"跳动"的灵魂）

- **弹簧阻尼**：柱子高度用 spring/damping 插值——攻击快、回落带一点回弹，而非生硬瞬变。这是节奏质感的关键。
- **波传播**：能量从中心向外像水波涟漪扩散（重拍 → 中心先涨 → 外圈延迟随后）。
- **堆芯脉冲**：低频重拍时中心核心瞬时爆亮 + 一圈光环外扩。
- **静息呼吸**：无强音时用噪声让全场缓慢起伏，永远微动。
- **频段分层**：低频 → 中心高度 & 堆芯；中频 → 中圈；高频 → 外圈细密抖动。
- **相机**：极缓慢自动环绕 + 上下正弦浮动，无限循环。

---

## 8. 音频映射与接口

- 频谱抽象成约 **64 段**，从中心到外圈一环一段（环编号 → 频段编号）。
- **本期**：`SimulatedAudioSource` 用「多正弦 + 噪声」合成出像真音乐的假频谱（有节拍、有起伏）。
- **接口契约**（视觉层只依赖它，换数据源不改视觉）：
  - `getSpectrum(): Float32Array(64)` —— 归一化 0..1 的频段能量
  - `getLevels(): { bass, mid, treble }` —— 三段聚合能量，驱动堆芯/相机/整体律动
  - `update(dt)` —— 推进内部状态
- **未来**：实现 `WebAudioSource`（Web Audio `AnalyserNode` 真实 FFT），其余视觉代码零改动。

---

## 9. 性能与壁纸适配

- 用 **`InstancedMesh`** 一次性绘制上千根柱子（每根柱子=一组实例方块），性能稳。
- 目标 60fps；壁纸可锁 30/45fps 省电。
- **页面不可见时降帧 / 暂停**（`visibilitychange`），后台长跑不空耗。
- 全屏自适应（`resize`）；支持 Retina，但限制 `pixelRatio`（≤2）省电。
- 深色为主，OLED 友好。
- **三档画质**：低/中/高（柱数、Bloom、粒子数、像素比）。

## 10. 技术架构 / 模块拆分

每个模块单一职责、接口清晰、可独立理解与替换：

| 模块 | 职责 | 依赖 | 接口 |
|---|---|---|---|
| `config.js` | 集中所有可调参数 | 无 | 导出 `CONFIG` 对象 |
| `audioSource.js` | 频谱数据源（本期模拟，未来真实） | 无（模拟）/ Web Audio（未来） | `getSpectrum()` / `getLevels()` / `update(dt)` |
| `sceneSetup.js` | scene / camera / renderer / fog / 背景 / 灯光 | three | 返回 `{ scene, camera, renderer }` |
| `pillarField.js` | 建网格 InstancedMesh，频谱→目标高度→弹簧阻尼→更新实例 | three, config | `build()` / `update(spectrum, levels, dt)` |
| `core.js` | 中心堆芯发光体 + 光环涟漪 + 重拍脉冲 | three, config | `build()` / `update(bass, dt)` |
| `starfield.js` | 背景星点粒子 | three | `build()` / `update(dt)` |
| `cameraRig.js` | 相机缓慢环绕 + 浮动 | three, config | `update(dt)` |
| `postfx.js` | Bloom + 暗角后期 | three EffectComposer | `render()` / `resize()` |
| `main.js` | 装配各模块、rAF 主循环、resize、可见性降帧 | 以上全部 | 入口 |

建议目录：`index.html` + `src/`（上述模块）+ `assets/`（如需）。

## 12. 本期范围 / 非目标

**本期做（In scope）**
- Three.js 全屏 3D 场景：曲面分段方块柱阵 + 中心堆芯 + 星点 + 雾 + Bloom + 相机漂移
- 模拟音频驱动跳动（含静息呼吸、波传播、堆芯脉冲）
- 全屏自适应、无缝循环、可见性降帧、三档画质
- 干净的 `audioSource` 接口，为接真实音频预留

**本期不做（Non-goals，后续）**
- 接入 macOS 系统真实声音（BlackHole / 录音权限 / Web Audio 真实 FFT）
- Plash 部署与开机自启的具体配置
- 设置/参数面板 UI
- 频率标签、播放器、歌词、品牌 logo 等 HUD 元素

## 13. 验收标准

- [ ] 浏览器全屏打开后铺满整屏，循环无缝、无明显起止
- [ ] 约 60fps（中档画质，普通 Mac）；后台不可见时自动降帧
- [ ] 柱阵随模拟频谱实时跳动，弹簧阻尼回落手感自然
- [ ] 中心堆芯随低频重拍爆亮并扩散光环
- [ ] 无音乐输入时仍有静息呼吸，画面不死
- [ ] 配色与参考图一致（深空蓝紫 + 薰衣草峰值），分段方块柱质感清晰
- [ ] 切换 `audioSource` 数据源不需改动视觉模块（接口隔离验证）
